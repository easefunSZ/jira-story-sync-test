## 📖 Original Description
All current validations and business rules are hardcoded. The team has proposed two approaches:
* * Option A: Full migration — migrate all existing hardcoded rules to a configurable framework
Option B: New rules only — keep existing hardcoded rules, build configuration tables only for new rules
There is currently insufficient information to assess the impact of having both approaches coexist. This Story aims to provide the data needed for a decision.
Time Box: Investigation should not exceed 2 working days.
Deliverables:
#
Deliverable
Description
1
Hardcoded Inventory
Identify all hardcoded items, categorized by type (business rules / validation rules / field rules), listing rule name, code location, and linked Jira Story
2
Effort Estimation
For each hardcoded rule, estimate the development effort, testing effort, and risk level required for configurability conversion
3
Option Comparison & Recommendation
Pros/cons comparison of Option A (full migration) vs Option B (new rules only), including effort, risk, maintenance cost, and a recommended approach
