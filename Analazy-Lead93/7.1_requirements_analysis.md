## 2. Key Inconsistencies & Conflict Points (RESOLVED)

### ⚠️ Conflict 1: Category Deletion & Resolution Behavior
* **Resolution**: The category deletion feature has been finalized as a **soft delete**. 
  * **Rule**: Deletion of a category is **strictly blocked** if templates are assigned to the category or any of its subcategories. Content Managers must manually reassign or delete templates beforehand.
  * **Cascade Rule**: Deleting a parent category automatically marks all its child subcategories as Deleted (soft delete). Deleted nodes are hidden from filters, navigation, and cannot be edited.
  * **Audit**: An audit log must record the deletion (who, when, ID, and Name).

### ⚠️ Conflict 2: Template Categorization (Many-to-Many vs. One-to-Many)
* **Resolution**: A template belongs to **exactly one Category**, but may belong to **multiple Subcategories** within that selected Category.
  * **Reassignment**: Dragging a template to a new Category moves it (not copies) and clears all previous subcategories. The user is prompted to select new subcategories belonging only to the target Category.
  * **Folder Tree**: In navigation trees, the template appears under each assigned subcategory within its category.

### ⚠️ Conflict 3: Workflow States & Role Definitions
* **Resolution**: Template editing locking rules apply identically to both `Draft` and `Published` states. Direct `Draft -> Published` transitions are supported.

## 3. Points of Clarification & Open Questions (RESOLVED)

### 🔍 Q1: Concurrency Lock Scope (LEAD-278)
* **Resolution**:
  1. The edit lock **only** applies when a user edits the template content body (WYSIWYG editor).
  2. Metadata editing (like title, category, or tags) is **not** affected by the lock and works independently.
  3. The lock is released on save and exit, user inactivity timeout, or session end.

### 🔍 Q2: Predefined Taxonomy vs. Free-Text Tags (LEAD-300)
* **Resolution**: Tags must be selected **only** from a predefined lookup list grouped by Content Type, Trigger Event, Lifecycle Stage, and Financial Need. Free-text entry is prohibited. Required tag groups must contain at least one tag.

### 🔍 Q3: Legacy Template Migration Strategy (LEAD-328)
* **Resolution**: Developer-led database alignment script to parse 79 legacy templates, migrating HTML content and mapping target organizations to channels (PFA, AFD, DFA).

### 🔍 Q4: Versioning Depth & Visualization (LEAD-280)
* **Resolution**: Version history tracks both body content changes and metadata updates (e.g. Category/Subcategories changes). A version list with commit comments and timestamps is sufficient.

## 4. Architectural & UI Recommendations

1. **Category/Subcategory Database Mapping**: Implement a join table `iic_crm_campaign_template_subcategory` for the template-to-subcategory mapping, while keeping a single `category_id` in the main template table for parent category mapping.
2. **Category Governance**: Enforce category hierarchy depth limits (maximum depth <= 2) and sibling uniqueness checks directly in the backend `POST /api/v1/admin/categories` and `PUT` endpoints.
