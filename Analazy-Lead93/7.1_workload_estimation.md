# DAE Lead-93 Template Library Workload Estimation Report

This report provides a detailed front-end and back-end workload estimation for the **14 active stories** currently linked to the Jira Feature **LEAD-93**. The estimations are based on the project's technical stack:
- **Frontend**: React 17 + AntD 4.x + qiankun micro-frontend
- **Backend**: Spring Boot 3.2 + MyBatis-Plus + Redis + AWS S3

---

## 1. Estimation Methodology
This document presents the actual base workload estimates **without buffer time**.
*   Both frontend and backend estimates are displayed as actual raw values (decimals allowed).
*   No rounding or ceiling buffer logic is applied.

## 3. Key High-Effort Stories Breakdown

### 3.1 LEAD-278: Edit a Published template (Total: 3.5 PD)
* **Frontend (0.5 PD)**: 
  * Rich-text editor integration with customized dropdown variables insertion.
  * Concurrency lock polling mechanism (sending heartbeat API call to Redis every 5 minutes).
  * Enforcing read-only view state with lock banner when the template is locked by another content manager.
* **Backend (3.0 PD)**:
  * Redis distributed lock service (`lock:template:edit:{templateId}` using `SET NX` with 10-minute TTL).
  * Custom annotation `@RequireLock` and lock validation aspect (`LockAspect`).
  * Jsoup HTML sanitizer to prevent XSS.

### 3.2 LEAD-280: Template versioning (Total: 0.5 PD)
* **Frontend (0.0 PD)**:
  * Version history timeline and comparison uses existing components.
* **Backend (0.5 PD)**:
  * Reuses existing database schema and audit logger setup for `template_version_history`.
  * Snapshot creation on publication and copy back to `draft_*` fields.

### 3.3 LEAD-279: Manage Draft & Publish Workflow (Total: 3.0 PD)
* **Frontend (0.5 PD)**:
  * Display workflow status badges (`Draft`, `Preview`, `Published`).
  * Double-confirmation dialogs for Publish and Discard Draft actions.
* **Backend (2.5 PD)**:
  * State transition validation and database transaction control.
  * Dual-zone data architecture management (`draft_*` columns for workspace edit, `live_*` columns for adviser consumption).
