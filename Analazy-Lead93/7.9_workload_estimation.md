# DAE Lead-93 模板库后端工作量评估报告 (策略 A - 生产系统极简无锁版)

本报告提供了针对 **Lead-93 (Template Library Management & Governance)** 需求下 **13 个活跃 Story** 的后端开发估时。本估时已根据**策略 A (最接近斐波那契数对齐)** 进行取整折中（所有故事卡总估点限制在 **1、2、3、5、8** 内）。

本估算已彻底排除了所有前端开发工时与人工运维订正工作量。特别地，本版本**完全沿用实际生产系统中的现有设计方案**，即直接在生产表 `iic_msg_email_config`（模板主表）和 `iic_msg_email_config_version`（版本表，通过 `version_status` 标识 0-草稿/预约、1-生效、2-过期）的基础上进行扩展和控制增强，无过渡的 `iic_msg_email_template_meta` 元数据表。

**移除了并发编辑排他锁功能，以及发布阶段的双写事务回滚和乐观锁防覆盖，完全保持生产系统现有的无锁无事务重构并发状态**；针对 **LEAD-328 (历史数据迁移)**，确认直接在数据库中编写一次性批量更新 SQL 脚本进行迁移清洗。

---

## 一、 估算原则与基础说明
1. **工作量单位**：人天 (Person-Days, PD)。
2. **估算值精度**：故事级估时完全对齐到敏捷标准斐波那契序列：**1、2、3、5、8** 人天。
3. **技术栈基础**：Spring Boot 3.x + MyBatis-Plus + 生产版本表现有状态机机制。

---

## 二、 模板库后端工作量评估汇总表 (13 Stories)

| 故事卡号 (Story ID) | 故事标题 (Story Title) | 后端估时 (BE PD) | 核心技术实现要点与流程关注点 |
| :--- | :--- | :---: | :--- |
| **LEAD-293** | Create a category/ subcategories | **3** | 2级树形分类 CRUD。后端支持深度校验与同级重名校验；包含 DDL 审批流程。 |
| **LEAD-307** | Delete a category/subcategory | **2** | 分类删除校验。仅允许空目录软删除，递归级联；包含删除审计日志。 |
| **LEAD-276** | Template Reassignment | **2** | 跨分类重分配主逻辑；DBA 审核多对多关系映射表更新。 |
| **LEAD-306** | Create a new template | **2** | 新建模板工作流。在生产表 `iic_msg_email_config` 新建并向 `iic_msg_email_config_version` 插入草稿版本。 |
| **LEAD-301** | Assign & Edit Category & subcategory | **2** | 模板与分类/子分类的多重分配。**合并 DDL** 优化。 |
| **LEAD-300** | Select /Assign and Edit Tags | **2** | 4大维度标签属性绑定。**合并 DDL** 打包审批。 |
| **LEAD-278** | Edit a Published template | **1** | 富文本编辑。**无锁现有设计**：编辑在线模板时，自动在 `config_version` 写入状态为草稿（`version_status = 0`）的新版记录，以进行发信隔离。无需并发锁。 |
| **LEAD-279** | Manage Draft & Publish Workflow | **1** | 草稿与发布工作流。**极简方案**：控制 `config_version` 的状态变更（0草稿 -> 1生效，原生效改 2过期）。已剔除双写全局回滚与版本乐观锁，丢弃时直接物理清理草稿版。 |

| **LEAD-277** | Template data model & validation | **3** | 校验框架与主表分类列外键扩容。手写 `PublishValidator` 校验链，大括号正则安全性校验。 |
| **LEAD-326** | Template Preview | **1** | 模板预览与附件分发。仅从 `config_version` 获取草稿数据运行 Dry-run 预校验 API。 |
| **LEAD-327** | Search and Filter Templates | **3** | 多维检索，处理多选标签 AND EXISTS 高级 SQL 并交集过滤。 |
| **LEAD-328** | Data alignment/Template Migration | **1** | **一次性 SQL 跑批**：编写批量迁移/清洗 SQL 完成 79 个模板历史数据分类与标签初始化。 |
| **合计** | **13 个 Stories** | **25** | **项目后端开发与联调部署总工作量：25 人天** |

## 三、 排期与并行开发建议
建议配置 **2 名后端工程师** 并行推进项目，日历排期计划如下：

1. **第一阶段：分类目录树与数据映射 (第 1 周)**
   * **任务**：LEAD-293 (分类树) + LEAD-301 (分类绑定) + LEAD-300 (标签绑定) + LEAD-276 (模板移动)。
   * **关注点**：提交第一批 DDL 变更单（合并后的 2 个 CR 申请单）给 DBA 审批，预置 4 大维度标签静态数据。
2. **第二阶段：模型校验与模板流程 (第 2 周)**
   * **任务**：LEAD-277 (主表外键 DDL 及校验器) + LEAD-306 (创建模板) + LEAD-279 (状态机控制) + LEAD-326 (预校验API)。
   * **关注点**：实现新版本插入时的安全拦截，保证发信系统级联联调。
3. **第三阶段：编辑下线与搜索 (第 3 周)**
   * **任务**：LEAD-278 (草稿版本自动生成逻辑) + LEAD-296 (删除停用) + LEAD-307 (分类删除) + LEAD-327 (多维检索)。
   * **关注点**：编写草稿版本保存及版本历史校验单测，处理多选标签 EXISTS 并交集的高性能过滤 SQL。
4. **第四阶段：迁移与级联发信联调 (第 4 周)**
   * **任务**：LEAD-328 (批量 SQL 迁移演练) + 线上级联发信系统联调。
   * **关注点**：在 SIT/UAT 环境下由 DBA 演练并执行存量 79 个历史数据清洗及元数据表初始化 SQL。
